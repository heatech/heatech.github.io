为什么hexo

发表于 2015-09-05   |   分类于 coding   |  

hexo是一个基于nodejs的强大博客系统。令人骄傲的是其作者是台湾学生，所以系统对于中文字符支持的比较好。从 github 上查到最早的一次提交与2012年9月10日，到现在不过3年时间，其发展速度很快。2015年3月5日发布了最新的3.0版本，其插件、主题也有很多。

准备

如果想使用hexo写博客，至少要了解以下一点点技术：

  • nodejs，一切自动化命令都依赖与他，世界的根基。
  • npm，安装nodejs时会自动安装好这个。nodejs的包管理命令，使用它可以安装上hexo。
  • hexo，使用这个命令可以完成初始化博客、新建、发布等操作。
  • Markdown，对于程序猿来说，不懂Markdown就像不会使用Word，无法与人交流。
  • github，最后你的博客要发布于此，各种主题也是从这里clone下来的。
  • ejs，如果你想修改主题，或者新建主题里的网页模板，懂得Embedded JavaScript templates。
  • Swig，好吧，有的主题还是使用的Swig当作模板
  • stylus，如果你想修改主题，或者新建主题里的样式，需要懂得stylus.

看到这么多技术，是不是感受到一股弄弄的geek风呢，绝对值得你把玩上几天。

为什么hexo

说到这里，还没有切入正题，什么要使用hexo当作博客系统呢。其实上面已经谈到了一部分，当然就是：hexo是为geek准备的。另外，由于其本质是发布后生成纯静态网页，所以会附带更多的好处，借用equation85的话说：

  • 可以在各种文字编辑器中通过Markdown或者Textile来写博文。
  • 可以在本地预览博客。
  • 写作无需Internet连接。
  • 可以通过Git进行发布。
  • 可以在静态Web服务器上构建博客站点。
  • 可以通过Github Pages服务免费构建博客。
  • 无需数据库。

另外，吸引我的还离不开众多为hexo定制的主题，例如：

  • Pacman
  • Jacman
  • NexT

这些主题不仅样式优雅，并且功能强大。特别是由于这些主题的作者都是国人，所以默认集成了像百度统计、多说评论等功能。很时髦。

使用Markdown书写有很大好处，现在已经有很多基于Markdown的独立博客系统，例如简书。值得一提的是，简书在iOS客户端的字体样式很好看。

独立精神

很赞同WuChong在 Hello独立博客 中谈到的：

  1. 独立，自由，简洁。不会像某些平台上的博客一样，广告乱飞。
  2. 够酷。能有一个专属自己的域名。即使是heatech.github.com，那也不是我的，那是在别人的地上种菜。一定要heatech.me，那才是自家的地（虽然本质上仍然不是我的，但门面上是）。
  3. 回馈精神。就像开源软件，我们不断地从开源社区获得那么多的便利与知识，当然也要回馈社区。
  4. 成长的记录。即时你现在分享的是别人耻笑的水文，那又如何。在分享的过程中，已经远远超过那些人了。

部署教程

最后，对于hexo安装使用教程网上也很多，这里我就不重复造轮子了，以下是推荐的教程。

  • 如何搭建一个独立博客——简明Github Pages与Hexo教程，比较全面的介绍github和hexo的部署教程。另外博主的其他文章也很值得一读。
  • hexo你的博客，介绍了目录的基本结构，和一些插件和工具。
  • 安装和使用NexT主题
  • Zippera’s Hexo系列教程
    • 主要是针对light主题
    • 可以将hexo部署到国内的 gitcafe 或者 STDYUN 服务。甚至在gitcafe上创建多个page服务
  • Hexo UML插件
  • 不蒜子，静态网站访问量计数
  • 部署 Hexo 静态博客至七牛
  • Hexo 优化与定制(二)，添加搜索

使用经验

  1. hexo d -g命令等于hexo generate + hexo deploy
  2. hexo server --debug可以开启调试信息，观察变化。目前发现.md文件保存的时候hexo server都会自动更新。
